# When / why to launch (operator tips)

Use this as a pre-flight checklist. Fill in **your** numbers — never copy another artist's limits.

## Launch only when

1. **Creative is review-ready** — product is clear in the first seconds (brand + offer). Music-only with no spoken product often fails review.
2. **Companion + CTA match** — audio message and click destination agree (e.g. LISTEN_NOW → track or listen hub).
3. **Landing works** — if you drive off-platform, the URL loads on mobile.
4. **Budget floor met** — ad set can hit platform minimums in the market currency.
5. **Kill rule written** — e.g. pause if cost-per-listener (or your success metric) exceeds X after Y spend.
6. **Delivery plan** — OFF until APPROVED, then ON only after you confirm.

## Hold / fix first when

- Song-forward creative with no VO/product framing
- Missing companion for AUDIO
- Unverified geo / audience too small (run audience estimate)
- No success metric or kill rule
- Account billing / API verify not ready for write mode

## Test design

- One major variable per cell (audience **or** message **or** format)
- Preserve baseline conditions when comparing
- Treat metrics as observations, not causal proof

## After go-live

- Check pacing within 24–48h
- Do not raise budget on day-1 noise alone
- Log learnings with evidence + confidence (LOW/MEDIUM/HIGH)
